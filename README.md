# Курсовая работа: прогнозирование эффективности химических соединений

Это репозиторий с курсовой работой по классическому машинному обучению. В работе используются данные по химическим соединениям против вируса гриппа. Нужно было построить модели для трёх регрессионных задач и четырёх задач классификации.

Структура репозитория:

```text
flu_ml_coursework/
├── README.md
├── requirements.txt
├── coursework_data.xlsx
├── eda.py
├── analytical_report.pdf
└── models/
    ├── regression_ic50.py
    ├── regression_cc50.py
    ├── regression_si.py
    ├── classification_ic50_median.py
    ├── classification_cc50_median.py
    ├── classification_si_median.py
    └── classification_si_gt8.py
```

## Что лежит в файлах

- `coursework_data.xlsx` — исходные данные.
- `eda.py` — разведочный анализ данных. Скрипт проверяет размер таблицы, пропуски, дубликаты, константные признаки, распределения целей, выбросы и корреляции.
- `models/regression_ic50.py` — регрессия для `IC50, mM`.
- `models/regression_cc50.py` — регрессия для `CC50, mM`.
- `models/regression_si.py` — регрессия для `SI`.
- `models/classification_ic50_median.py` — классификация `IC50, mM` выше медианы обучающей выборки.
- `models/classification_cc50_median.py` — классификация `CC50, mM` выше медианы обучающей выборки.
- `models/classification_si_median.py` — классификация `SI` выше медианы обучающей выборки.
- `models/classification_si_gt8.py` — классификация `SI > 8`.
- `analytical_report.pdf` — итоговый аналитический отчёт.

Отдельных обученных `.joblib`-моделей в репозитории нет. Для задания они не требуются, а на GitHub такие бинарные файлы только утяжеляют проект. Все модели можно переобучить запуском соответствующих скриптов.

## Как запустить

Я запускал проект на Python 3.12. Должно работать и на Python 3.10+, если установлены зависимости из `requirements.txt`.

Сначала нужно перейти в папку проекта:

```bash
cd flu_ml_coursework
```

Создать виртуальное окружение можно так:

```bash
python -m venv .venv
source .venv/bin/activate
```

На Windows команда активации будет другой:

```bash
.venv\Scripts activate
```

Дальше установить зависимости:

```bash
pip install -r requirements.txt
```

## Запуск EDA

```bash
python eda.py
```

Скрипт напечатает основные выводы в консоль и сохранит графики в папку `outputs/eda/`. Эта папка создаётся автоматически при запуске и не нужна для первоначальной сдачи.

## Запуск моделей

Каждая задача запускается отдельно. Например:

```bash
python models/regression_ic50.py
```

Полный список команд:

```bash
python models/regression_ic50.py
python models/regression_cc50.py
python models/regression_si.py
python models/classification_ic50_median.py
python models/classification_cc50_median.py
python models/classification_si_median.py
python models/classification_si_gt8.py
```

Каждый скрипт сам загружает данные, удаляет из признаков целевые колонки `IC50, mM`, `CC50, mM`, `SI`, делает train/test split, подбирает гиперпараметры через `GridSearchCV` и печатает таблицу результатов. Дополнительно таблица сохраняется в `outputs/metrics/`.

На моём запуске полный прогон всех семи задач занимал несколько минут. Самыми долгими обычно оказываются ансамбли деревьев.

## Важные методические решения

Главный риск в этой задаче — утечка данных. `SI` рассчитывается на основе `IC50` и `CC50`, поэтому во всех задачах из признаков удаляются все три целевые колонки сразу. Это сделано даже тогда, когда предсказывается только одна из них.

Для задач классификации по медиане порог считается только по обучающей выборке. Это важно, потому что медиана всей таблицы уже содержит информацию из тестовой части.

Препроцессинг находится внутри `Pipeline`: заполнение пропусков и масштабирование обучаются только на train и внутри кросс-валидации, а не на всей таблице сразу.

## Если что-то не запускается

1. Проверьте, что файл с данными называется именно `coursework_data.xlsx` и лежит в корне проекта.
2. Запускайте команды из корня проекта, а не из папки `models`.
3. Если не читается Excel-файл, чаще всего не установлен `openpyxl`: `pip install openpyxl`.
4. Если долго обучаются модели, это нормально: в каждой задаче сравниваются несколько моделей и подбираются параметры.

## Краткий итог

В работе решены все семь задач из задания. Лучше всего из целевых переменных предсказывается `CC50`, хуже всего — прямая регрессия `SI`. Для практического отбора перспективных соединений более устойчивой выглядит классификация `SI > 8`, а не попытка точно предсказать значение `SI`.
